# ARTLINE Rich Studio

Платформа для створення, перевірки, версіонування та зберігання rich-контенту для карток товарів ARTLINE.

Вставляєте посилання на товар — система читає сторінку, збирає підтверджені факти, генерує готовий rich-HTML під десктоп і мобільну версію в потрібних мовах, створює product-in-use зображення з реального фото товару та віддає результат на перевірку.

---

## Основні можливості

### Генерація контенту

- **Старт з одного посилання.** Назва, категорія та артикул визначаються автоматично з аналізу сторінки.
- **Факти замість вигадок.** Джерело правди — дані самої сторінки: JSON-LD, таблиці характеристик, `<dl>` і spec-блоки з HTML, доповнені AI-аналізом. Модель не має права вигадувати специфікації, гарантії чи сумісність.
- **Дві версії макета.** Desktop (до 1240px) і Mobile (480px) генеруються окремо, з композицією під кожен viewport.
- **До 10 мов.** Українська та Російська — за замовчуванням, Польська — швидким вибором, решта додаються за ISO/BCP-47 кодом. Для кожного viewport створюється один майстер-макет, інші мови отримуються перекладом **лише текстових вузлів** — тому структура, CSS, порядок секцій і зображення повністю збігаються між мовами.

### Зображення

- **Реальне фото товару.** Головне фото визначається з галереї, `srcset` і zoom-посилань, варіанти одного кадру об’єднуються, обирається найкраща роздільність.
- **Product-in-use генерація.** Hero та Feature створюються через OpenAI Images Edit **на основі реального фото** зі збереженням геометрії, матеріалів, портів і брендування. Окремі Hero для десктопа (`1536×1024`) і мобільної (`1024×1536`).
- **Безпечний відкат.** Якщо перевіреного референсу немає або редагування не вдалося — використовується справжнє фото товару, вигаданий товар не генерується ніколи.

### Стилі

- **`ARTLINE Base`** — вбудований керований стиль: єдиний дизайн-код, система контентних «острівців», контроль контрасту й доступності, структура з шести секцій, корисний для SEO та генеративних пошукових систем текст. Оновлюється автоматично при запуску API.
- **Власні стилі:** AI-генератор стилів, покращення через AI, оцінка якості, історія версій, автозбереження, видалення. Проєкти видаленого стилю переприв’язуються до базового.

### Контроль якості та рев’ю

- **AI-критики:** HTML-структура, фактологія, доступність, маркетинг — з підсвічуванням зауважень і можливістю перезапуску. Це евристичні автоперевірки, які не замінюють ручний контроль.
- **Ручна перевірка:** чеклист, коментар, рішення «Схвалити» / «Запросити зміни» за ролями.
- **Версіонування:** редактор HTML зі збереженням нових версій, повна історія етапів проєкту.

### Керування й облік

- **Черга:** пауза, скасування та відновлення реально зупиняють і повертають worker; живий прогрес і таймер виконання.
- **Вартість:** поетапна розбивка (аналіз товару, генерація тексту, зображення) з підсумком, облік токенів і зображень.
- **Медіатека:** пошук і фільтри за джерелом, роллю, проєктом; сортування за датою, роздільністю, вартістю.
- **Експорт:** ZIP з останніми HTML-версіями, фрагментами для вставки, зображеннями, стилем і `manifest.json`.
- **Доступ:** ролі `admin` / `editor` / `reviewer` / `viewer`, пряме створення користувачів і реєстрація за запрошенням.

### Безпека

- Увесь rich-HTML санітизується на сервері (вирізаються `script`, обробники `on*`, `javascript:`-посилання).
- Захист від SSRF: зовнішні URL перевіряються, приватні та loopback-адреси блокуються.
- Обмеження частоти спроб входу, гарантія щонайменше одного активного адміністратора, JWT.

## Як це працює

```
Посилання на товар
   ↓  scrape    читання сторінки
   ↓  extract   факти: назва, бренд, артикул, категорія, характеристики
   ↓  images    вибір реального фото → Hero / Feature через Images Edit
   ↓  content   майстер-макет на viewport → переклад текстових вузлів
   ↓  critics   автоперевірки якості
   ↓  review    ручне схвалення
Готовий rich-HTML + ZIP
```

Стек: FastAPI + Celery + PostgreSQL + Redis, статичний фронтенд на nginx, розгортання через Docker Compose.

## Швидкий старт

```bash
cp .env.example .env
nano .env
chmod +x deploy-truenas.sh
./deploy-truenas.sh
```

- WebUI: `http://SERVER_IP:3000`
- API-доки: `http://SERVER_IP:8000/docs`
- Перевірка: `curl http://127.0.0.1:8000/health` → `{"status":"ok","version":"12.0"}`

Повний runbook (деплой готовими образами з GHCR, відкат, бекап, міграції) — **[docs/DEPLOY.md](docs/DEPLOY.md)**.

## Обов’язкові змінні

```env
POSTGRES_DB=richstudio
POSTGRES_USER=richstudio
POSTGRES_PASSWORD=replace-me
DB_SCHEMA=richstudio_v11_2
REDIS_URL=redis://redis:6379/0
JWT_SECRET=replace-with-long-random-secret
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=replace-me
OPENAI_API_KEY=
```

Без `OPENAI_API_KEY` система працює в офлайн-режимі на вбудованому детермінованому шаблоні.

Схема таблиць сумісна з `richstudio_v11_2` — оновлення зберігає наявні проєкти, стилі та користувачів. Нові колонки додаються ідемпотентними міграціями при старті.

## Перевірки

```bash
./scripts/validate.sh
PYTHONPATH=apps/api pytest -q tests
```

`validate.sh` проганяє компіляцію, перевірку синтаксису JS, smoke-перевірки та конфіг Docker Compose. Регресійні тести покривають вибір головного фото, відновлення некоректного JSON-LD, збереження DOM при перекладі та відсутність `<h1>` у rich-контенті.
