# ARTLINE Rich Studio v11.8

Внутрішня платформа для створення, перевірки, версіонування та зберігання rich-контенту.

## Що змінено у v11.8

Безпека та стабільність:

- Увесь rich-HTML (згенерований, збережений вручну та preview стилів) проходить санітизацію на сервері: вирізаються `script`, `style`, `iframe`, обробники `on*` та посилання `javascript:`. Це закриває збережений XSS у перегляді та редакторі.
- Додано захист від SSRF: власні Hero/Feature URL та зовнішні зображення в архіві перевіряються — приватні, loopback і link-local адреси блокуються.
- Вхід має обмеження частоти спроб, адміністратор не може деактивувати або розжалувати сам себе (гарантується щонайменше один активний адмін), JWT переведено з застарілої `python-jose` на `PyJWT`.

Виправлено:

- Пауза та скасування тепер реально зупиняють worker між етапами, а не лише змінюють статус у базі.
- Повторний запуск блокується не тільки під час `processing`, а й у стані `queued` — подвійний клік більше не створює дубль генерації.
- Прибрано дубльований маршрут версій стилю; тихий відкат на шаблон під час помилки OpenAI тепер видно в історії проєкту.

Додано:

- Видалення проєкту разом з версіями, зображеннями, критиками й історією (кнопка у робочій області).
- Звіти AI-критики (HTML, фактологія, доступність, маркетинг) відображаються у вкладці «Перевірка» з можливістю перезапуску.
- Кнопки керування чергою (Пауза / Скасувати / Відновити) у робочій області.

Прибрано (мертвий код): невикористаний backend без UI — brands, knowledge, playground, compare, workflows, publish targets, benchmark, analytics і відповідні таблиці; застарілі legacy-константи промптів у `main.py`.

## Що додано у v11.7

- Додана єдина система контентних «острівців»: текст більше не плаває на порожньому полотні, а розміщується на узгоджених світлих, м’яких або темних поверхнях.
- Hero отримав компактний напівпрозорий темний острівець під текстом без поділу першого блоку на дві колонки.
- Key Benefits, Core Feature, Use Scenarios та Buyer Confidence отримали обов’язкові поверхні, однакові відступи, радіуси й межі.
- Прибрані голі текстові колонки та надмірні порожні проміжки між блоками.

## Що додано у v11.6

- Оновлений керований `ARTLINE Base`: єдиний сучасний дизайн-код, SEO/GEO-корисна структура, product-in-use Hero та Feature, сфокусований на головній підтвердженій перевазі.
- Вбудований базовий стиль автоматично отримує нову версію при запуску API; користувацькі стилі не перезаписуються.
- Гнучкий вибір до 10 мов: готові пресети плюс власні ISO/BCP-47-подібні коди під час створення та повторного запуску.
- Медіатека отримала пошук, фільтри за джерелом, роллю і проєктом, а також сортування за датою, роздільністю, вартістю, типом і назвою проєкту.
- Виправлена сумісність датованих `gpt-image-2-*`: параметр `input_fidelity` більше не надсилається моделям GPT Image 2.

## Що було додано у v11.5

- Новий системний стиль `ARTLINE Base` автоматично створюється у розділі стилів і стає базовим для нових проєктів.
- У списку та робочій області проєкту показується назва використаного стилю.
- Повторна генерація відкриває діалог вибору стилю; попередні HTML-версії залишаються в історії.
- Картки проєктів поводяться як кнопки: текст не виділяється, додана клавіатурна навігація.
- URL вихідного товару всередині проєкту відкривається в новій вкладці.
- Існуючі PostgreSQL enum автоматично оновлюються значеннями Review, тому `approve` і `request_changes` працюють без видалення бази.
- Проєкт можна завантажити як ZIP з останніми HTML-варіантами, фрагментами, зображеннями, стилем і manifest.
- Редактор існуючого стилю автоматично зберігає кожну зміну після короткої паузи та показує стан збереження.

## Що виправлено у v11.4

- Для кожного viewport генерується один майстер-макет; RU/UA/PL отримуються перекладом лише текстових вузлів, тому структура, CSS, секції та зображення збігаються.
- Hero генерується окремо для Desktop (`1536×1024`) і Mobile (`1024×1536`) з різними правилами композиції та спільним перевіреним фото товару.
- Перегляд отримав світле полотно ARTLINE з окремою шириною Desktop/Mobile, а HTML-редактор переведено на світлу схему.
- На сторінці проєктів доступні два режими — картки та компактний список; вибір зберігається у браузері.
- HTML-сутності у назві товару декодуються, тому замість `&#039;` відображається нормальний апостроф.

## Що виправлено у v11.3

- Реальне фото товару визначається з `Product.image`, галереї товару, `srcset` і zoom/modal-посилань.
- Варіанти 220/600/1400/original одного кадру об’єднуються за ідентичністю; перевагу має головне фото, а не наступна інфографіка.
- Прибрано помилкове обмеження 20 КБ: оптимізовані WebP-фото більше не відкидаються через малий розмір файлу.
- Некоректний JSON-LD з зайвою закривною дужкою відновлюється, тому назва й головне фото фактичної сторінки ARTLINE розпізнаються правильно.
- Перевірений референс зберігається локально як `product-reference.png` і саме він передається в OpenAI Images Edit.
- Для підтримуваних GPT Image моделей запитується висока точність збереження вхідного зображення.
- Якщо референсу немає або редагування завершується помилкою, система залишає реальне фото й не генерує вигаданий товар.
- Медіатека окремо фільтрує AI-зображення, вихідні фото та власні завантаження; у картці видно вихідний референс.
- Preview має власну стабільну прокрутку і не скидається live-оновленням кожні 1,8 секунди.
- Виправлено «Відкрити окремо» та сортування за полями `created_at` / `finished_at`.
- Повторна генерація зберігає попередні HTML-версії, а не видаляє історію.
- Ініціалізація PostgreSQL захищена advisory lock; worker чекає готовності API, тому повторний запуск не створює гонку схеми.
- Серверне збереження HTML-версій додатково захищене блокуванням від одночасних записів.
- Rich-контент ніколи не містить `<h1>`: Hero та заголовки розділів використовують `<h2>`, картки — `<h3>`, а випадковий `<h1>` у відповіді моделі автоматично нормалізується перед збереженням.

## Основні можливості

- Україномовний WebUI у дизайн-системі ARTLINE з офіційним логотипом.
- Автоматична назва проєкту після аналізу сторінки товару.
- Динамічний список OpenAI-моделей з можливістю ввести model ID вручну.
- Hero/Feature промпти зберігаються лише у стилях і не є обов’язковими.
- AI Style Generator, покращення стилю, оцінка якості та історія версій.
- Live progress, RU/UA/PL, Desktop/Mobile, HTML-версії, review та облік використання.
- Пряме створення користувачів і реєстрація за запрошенням.

## Швидкий запуск на TrueNAS

```bash
cp .env.example .env
nano .env
chmod +x deploy-truenas.sh
./deploy-truenas.sh
```

WebUI: `http://SERVER_IP:3000`

API: `http://SERVER_IP:8000/docs`

Перевірка: `curl http://127.0.0.1:8000/health`

Очікувана відповідь:

```json
{"status":"ok","version":"11.8"}
```

## Обов’язкові змінні

```env
POSTGRES_DB=richstudio
POSTGRES_USER=richstudio
POSTGRES_PASSWORD=replace-me
DB_SCHEMA=richstudio_v11_2
REDIS_URL=redis://redis:6379/0
JWT_SECRET=replace-with-long-random-secret
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=replace-me
OPENAI_API_KEY=
```

Формат таблиць не змінювався, тому оновлення сумісне зі схемою `richstudio_v11_2` і зберігає наявні проєкти, стилі та користувачів. Для повністю чистої інсталяції можна вказати іншу нову схему вручну.

## Перевірки

```bash
./scripts/validate.sh
PYTHONPATH=apps/api pytest -q tests
```

Регресійні тести перевіряють вибір головного фото навіть тоді, коли правильний WebP-файл менший за 20 КБ, вибір high-resolution варіанта того самого кадру, відновлення некоректного JSON-LD та повну відсутність `<h1>` у rich-контенті.
