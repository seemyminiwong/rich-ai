# A/B: Gemini проти gpt-image-1 на реальних товарах

Мета: вирішити, яку модель ставити у пресет «Стандарт». Порівнюємо не бенчмарки, а те, що болить: чи зберігає модель зовнішній вигляд товару (корпус, шильдики, порти) і чи влучає Feature-зображення у текст секції Feature.

## Товари (5 категорій з реальної історії)

1. Генератор дизельний Qube QFED10000XE-3 — великий корпус, жовтий, багато наліпок
2. Система зберігання енергії Solarverse SV-1SV6K1-LES5.1K1 — два блоки, екран
3. 3D принтер Bambu Lab H2S — складна геометрія, скло
4. 3D сканер Creality CR-Scan Otter Lite — дрібний предмет
5. Будь-який товар з дрібним текстом на корпусі — найжорсткіший тест на логотипи

## Процедура (на кожен товар — два прогони)

1. Проєкт вже існує → «Повторити генерацію», галку «Використати наявні зображення» **зняти** (нам потрібні нові картинки).
2. Прогін A: модель зображень `gpt-image-1`, якість `medium`.
3. Прогін B: модель `gemini-2.5-flash-image` (безкоштовний тир — 500 зображень/день).
4. Мови можна лишити одну (ua) — текст нам не потрібен, це здешевлює прогін.

Орієнтовна вартість повного раунду: A ≈ $0.21/товар, B ≈ $0 (тир) — разом ~$1.1 на 5 товарів.

## Оцінка (по кожному зображенню, 1–5)

| Критерій | Що дивитись |
|---|---|
| Ідентичність товару | Корпус, пропорції, колір, порти — це той самий предмет? |
| Логотипи й текст | Шильдики не перемальовані, текст не «поплив»? |
| Сцена | Середовище правдоподібне, товар не «вклеєний»? |
| Feature влучає | Зображення показує саме те, що написано в секції Feature? |
| Текст-зона Hero | Чи лишилося чисте місце під текстовий острів? |

## Рішення

- B ≥ A у більшості → у пресеті «Стандарт» лишається `gemini-2.5-flash-image` (він там уже перший кандидат), економія ~40% на проєкт.
- A > B помітно → поміняти порядок кандидатів у `PRESETS.std.image` в `apps/web/app.js` і, за бажанням, спробувати `gemini-3-pro-image-preview` як третій варіант ($0.134, все одно дешевше за high).
- Нічия → лишити як є: Gemini дешевший.
